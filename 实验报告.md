# 基于 EM（GMM）算法的用户行为分群与分析

| 项目 | 内容 |
|------|------|
| 数据来源 | UCI Online Retail |
| 算法 | EM + 高斯混合模型（GMM） |
| 实现 | Python / scikit-learn / Streamlit |

## 1 背景简介

### 1.1 算法产生原因

电商交易数据中，用户消费贡献高度集中：少数用户贡献大部分销售额，多数用户购买频次低或长期未复购。对用户行为进行分层，需同时刻画消费时效、频次与金额三个维度。**RFM 模型**以 Recency（最近消费）、Frequency（购买频次）、Monetary（消费金额）构成用户特征向量，是零售领域常用的用户分层框架。

在 RFM 特征基础上，聚类算法用于从数据中自动划分用户群体。**高斯混合模型（GMM）**将观测数据建模为 $K$ 个高斯分布的加权混合，参数估计采用**期望最大化（EM）算法**。GMM 可输出样本属于各分量的后验概率（软分配），在分量分布重叠时仍能提供概率化归属结果。

与 K-means 相比，K-means 假设各簇球形、等方差，样本仅作 0/1 硬分配；RFM 三维空间中，Frequency 与 Monetary 常呈相关，簇边界多为椭圆形。GMM 采用完整协方差矩阵（`covariance_type="full"`），可刻画特征间相关结构。本实验采用 UCI Online Retail 交易数据，构建 RFM 特征后使用 GMM+EM 完成用户分群，并实现基于 Streamlit 的在线判别程序。

### 1.2 算法基本原理

#### （1）高斯混合模型

设共有 $n$ 个用户样本，每个用户的 RFM 特征向量为 $\mathbf{x}_i \in \mathbb{R}^d$（本实验 $d=3$）。GMM 假设数据由 $K$ 个高斯分量混合生成，混合系数 $\pi_k \ge 0$ 且 $\sum_{k=1}^{K}\pi_k = 1$，则单个样本的边际概率密度为：

$$
p(\mathbf{x}_i) = \sum_{k=1}^{K} \pi_k \, \mathcal{N}(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)
$$

其中 $\mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}, \boldsymbol{\Sigma})$ 为均值 $\boldsymbol{\mu}$、协方差 $\boldsymbol{\Sigma}$ 的多元高斯分布。待估参数为 $\{\pi_k, \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k\}_{k=1}^{K}$。

#### （2）EM 算法迭代过程

由于存在隐变量（样本所属高斯分量），直接极大化对数似然较困难。EM 算法交替执行 E 步与 M 步：

```
┌──────────────────────────────────────────────────────────────────┐
│                      EM 算法（重复直至收敛）                         │
├──────────────────────────────────────────────────────────────────┤
│  E 步：固定 π, μ, Σ，计算后验责任度 γ[i,k]                          │
│  M 步：固定 γ，更新 π, μ, Σ，极大化期望完全对数似然                  │
└──────────────────────────────────────────────────────────────────┘
```

E 步责任度计算公式为：

$$
\gamma_{ik} = \frac{\pi_k \, \mathcal{N}(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)}{\sum_{j=1}^{K} \pi_j \, \mathcal{N}(\mathbf{x}_i \mid \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)}
$$

M 步根据 $\gamma_{ik}$ 加权更新混合系数、各分量均值与协方差。迭代在对数似然变化低于阈值或达到 `max_iter` 时终止。实验采用 `sklearn.mixture.GaussianMixture`，设置 `covariance_type="full"`、`n_init=10`（多次随机初始化取最优解）、`reg_covar=1e-6`（协方差对角正则，防止奇异）。

#### （3）与 RFM 特征的结合

原始数据为订单级明细，按 `CustomerID` 聚合为用户级 RFM：Recency 为距分析快照日最近消费的天数，Frequency 为窗口内独立订单数，Monetary 为窗口内消费金额合计。Frequency 与 Monetary 呈右偏分布，实验对二者作 $\log(1+x)$ 变换，以训练集 99 分位数为上界裁剪，再经 `StandardScaler` 标准化。变换后的三维矩阵输入 GMM，经 EM 训练得到各用户的簇归属及后验概率，并映射为群体编号。

### 1.3 名词定义

| 名词 | 定义 |
|------|------|
| **RFM** | Recency / Frequency / Monetary，分别从消费时效、频次、金额描述用户 |
| **EM** | Expectation-Maximization，含 E 步（计算责任度）与 M 步（更新参数） |
| **GMM** | Gaussian Mixture Model，$K$ 个高斯分布的加权混合模型 |
| **软分配** | 样本以概率向量 $(p_1,\ldots,p_K)$ 属于各簇，而非 0/1 硬标签 |
| **责任度 $\gamma_{ik}$** | E 步输出的后验概率，表示样本 $i$ 属于簇 $k$ 的概率 |
| **BIC** | Bayesian Information Criterion，模型选择指标，数值越小通常表示拟合与复杂度权衡更优 |
| **轮廓系数** | Silhouette Score，衡量簇内紧密与簇间分离，取值 $[-1,1]$，越大越好 |
| **统计周期** | 近 1 年、近半年、近 1 季度、近 1 个月四档时间窗口，各训练一套独立模型 |
| **群体编号** | 固定的第 1/2/3 类，对应核心高价值、流失风险、活跃潜力 |
| **canonical_id** | 统一群体编号，与 GMM 内部簇下标 `cluster`（0/1/2）解耦 |
| **clip_bounds** | 训练阶段拟合的分位裁剪上界，预测阶段复用，保证特征变换一致 |

## 2 算法应用

### 2.1 数据预处理

#### 2.1.1 数据来源

本实验采用 UCI **Online Retail Data Set**，为英国在线零售商 2010 年 12 月至 2011 年 12 月的交易记录。原始数据约 **541,909** 条，字段包括 InvoiceNo、StockCode、Quantity、UnitPrice、CustomerID、InvoiceDate。数据经 `scripts/download_data.py` 转换为 `data/online_retail.csv`。该数据集规模较大，含缺失值、退款单与极端订单，适用于 RFM 聚合与聚类实验。

#### 2.1.2 清洗规则

清洗逻辑见 `core/dataset.py` 中 `clean_data` 函数，规则如下：

1. **删除 CustomerID 为空**：无法关联用户的订单不能构建用户级 RFM。
2. **删除 Quantity ≤ 0 或 UnitPrice ≤ 0**：无效记录影响 log 变换与聚类稳定性。
3. **删除退款单**：发票号以 **C** 开头的记录为冲销单据，予以剔除。

经清洗后，有效交易约 **397,884** 条，可聚合为 **4,338** 名客户（全量口径）。上述规则保证 RFM 与 GMM 输入的有效性与统计假设。

#### 2.1.3 时间窗口与 RFM 构建

实验设置四个统计周期，以数据最晚交易日次日为快照日，分别向前截取 **365 / 182 / 91 / 30** 天，对应近 1 年、近半年、近 1 季度、近 1 个月，各周期独立训练 GMM。

按 `CustomerID` 聚合 RFM 特征如下：

| 特征 | 计算方式 | 含义 |
|------|----------|------|
| Recency | 快照日 − 最近一笔 InvoiceDate 的天数 | 值越小表示消费越近期 |
| Frequency | 窗口内 InvoiceNo 去重计数 | 购买频次 |
| Monetary | 窗口内 $\sum (\text{Quantity} \times \text{UnitPrice})$ | 消费金额合计 |

近 1 年窗口内保留 **382,502** 条交易、**4,261** 名用户，作为该周期 GMM 的 $n$ 个样本。

#### 2.1.4 特征工程与标准化

Recency、Frequency、Monetary 量纲与分布差异大；Monetary 呈长尾，极端大额订单若未处理，将主导方差估计。实验采取：

- 对 Frequency、Monetary 作 $\log(1+x)$ 变换；
- 以训练集 99 分位数为上界裁剪，裁剪参数 `clip_bounds` 与模型一并保存；
- 对三维特征作 Z-score 标准化。

训练与预测阶段复用同一套变换参数，保证特征空间一致。

### 2.3 算法伪代码

#### 算法 1：EM 训练 GMM（单统计周期）

```
输入: 标准化特征矩阵 X (n×d), 簇数 K, 最大迭代次数 T, 协方差正则 ε
输出: 混合系数 π, 均值 μ[1..K], 协方差 Σ[1..K]

1.  随机初始化 π, μ, Σ（n_init=10 次，保留对数似然最优的一组）
2.  repeat 至多 T 次:
3.      // E 步
4.      for each 样本 i = 1 .. n:
5.          for each 簇 k = 1 .. K:
6.              γ[i,k] ← π[k] · N(x[i] | μ[k], Σ[k] + ε·I)
7.          归一化 γ[i,·]，使 Σ_k γ[i,k] = 1
8.      // M 步
9.      根据 γ 更新 π[k]、μ[k]、Σ[k]
10. until 对数似然变化 < 阈值 或 达到 T
11. return π, μ, Σ
```

#### 算法 2：在线 RFM 判别（单用户）

```
输入: RFM 三元组 (R,F,M), GMM, Scaler, clip_bounds, segment_profiles
输出: 群体编号, 群体名称, 置信度, 各簇概率

1.  x_raw ← (R, F, M)
2.  x ← engineer_rfm_features(x_raw, clip_bounds)
3.  x_scaled ← Scaler.transform(x)
4.  proba[k] ← GMM.predict_proba(x_scaled), k = 1..K
5.  cluster ← argmax_k proba[k]
6.  segment_id ← CANONICAL_MAP[cluster]
7.  生成个体 RFM 解读文本
8.  return segment_id, 名称, max(proba), proba
```

批量判别对 CSV 每行重复步骤 1～8，输出结果表。

### 2.5 时间和空间复杂度

设原始交易条数为 $N$，聚合后用户数为 $n$，特征维数 $d=3$，簇数 $K=3$，EM 迭代次数为 $T$。

| 阶段 | 时间复杂度 | 空间复杂度 | 说明 |
|------|------------|------------|------|
| 数据清洗 + RFM 聚合 | $O(N)$ | $O(N)$ | 单次扫描与分组聚合 |
| log1p + 标准化 | $O(n)$ | $O(n)$ | |
| EM 单次迭代 | $O(nKd^2)$ | $O(nK + Kd^2)$ | full 协方差 |
| EM 总训练 | $O(T \cdot nKd^2)$ | 同上 | $n_{\text{init}}=10$ 为常数因子 |
| 单用户预测 | $O(Kd^2)$ | $O(K)$ | |
| 批量预测 $m$ 用户 | $O(mKd^2)$ | $O(m)$ | |

本实验 $d=3$、$K=3$ 为常数，EM 训练主导项为 $O(Tn)$。四周期串行训练耗时约为单周期的四倍。单用户预测为毫秒级。

### 2.6 算法边界

本算法边界分三层：**程序硬性约束**、**训练数据有效范围**、**结果解读范围**。

**（1）程序硬性约束**

在线判别要求 R、F、M **均 ≥ 0**，批量输入须含对应三列；任一为负则报错拒绝。程序**不设上界**，过大取值仍会返回结果，但可能属于外推或截断，不宜直接当作可靠结论。

训练数据须满足：CustomerID 非空，Quantity 与 UnitPrice > 0，InvoiceNo 非退款单；周期内用户数 $n \geq K+1$（$K=3$）。模型固定为三维 RFM、$K=3$ 的 GMM，不适用于无客户标识或纯类别特征场景。

**（2）训练数据有效范围**

输入 RFM 的统计窗口须与所选周期一致（30 / 91 / 182 / 365 天）。以近 1 个月模型为例，训练用户 RFM 大致落在：Recency 1～29 天、Frequency 1～46 次、Monetary 15～168470 元（中位数分别为 12、1、353.98）。**建议判别输入落在该周期 min～max 内**；四周期完整范围见 `model_meta.json`。

F、M 经 $\log(1+x)$ 后按训练集 99 分位截断（近 1 月约 F≤6 次、M≤7562 元）；R 仅标准化、不截断。超出训练 max 的 R 直接外推；超大的 F、M 被截断后再判别。

**（3）边界示例**

| 情形 | 输入示例 | 行为 |
|------|----------|------|
| 合法范围内 | (12, 1, 353.98) | 正常判别 |
| 违反硬性约束 | R = −1 | **报错拒绝** |
| 超出训练范围 | R = 50 或 M = 200000 | 仍返回结果，属外推或截断 |

另有一种**语义边界**：群体名称来自簇均值（如「流失风险」表示该类平均 R 偏大），个体标签来自最大后验概率。图 2-3 中 R=1 仍可能判为第 2 类，并非程序错误，解读时需结合个体 RFM 与软概率，不能只看群体名称。

## 4 结论

**主要结果。** 本实验在 UCI Online Retail 数据上打通了「清洗 → RFM 特征工程 → GMM（EM）训练 → 在线判别」全流程（见图 2-1），并为近 1 年、近半年、近 1 季度、近 1 个月四个统计窗口各训练一套 $K=3$ 模型。训练样本规模随窗口缩短而递减（4,261 → 1,607 人），四周期 BIC 分别为 4654.80、4708.31、3687.49、2746.24，近 1 个月最低，表明在较短窗口、较近期行为下，三簇 GMM 对当前样本的拟合相对更优。轮廓系数方面，近 1 个月达 **0.3106**（四周期最高），近半年仅 **0.1598**（最低），近 1 年与近 1 季度分别为 0.1814、0.1987，说明**统计窗口的选择会显著影响分群可分性**：以近期 30 天行为建模时，用户群边界更清晰；半年窗口则簇间重叠较多，分群解释力相对较弱。近 1 个月模型将 1,607 名用户划分为核心高价值、流失风险、活跃潜力三类（217 / 1,077 / 313 人），群体语义与 RFM 业务含义基本一致。

**应用效益。** 训练与判别分离后，业务人员只需输入与窗口口径一致的 RFM 即可获得群体归属与置信度，支持单用户与批量判别。GMM 后验概率可识别归属不确定的边界用户，其价值高于单纯硬标签；解读时应结合个体 RFM、所属统计周期及轮廓系数水平，不能仅看群体名称字面含义。实践中可优先采用近 1 个月或近 1 季度模型开展短期运营触达，对长周期留存分析则宜结合近 1 年模型并谨慎解读边界样本。

**不足与展望。** 本实验固定 $K=3$，未系统搜索最优聚类数；特征限于 RFM 三维，轮廓系数整体处于中等偏低水平（最高 0.3106），分群结果宜作行为分层参考而非绝对分类。近半年窗口指标偏弱，提示单一时间窗口难以同时兼顾长期价值与短期活跃。后续可在 BIC 驱动下搜索 $K$、对比多窗口融合或分层建模，扩展品类、渠道等行为特征，并结合 A/B 实验验证分群对转化与留存的真实贡献。

def fit_gmm(X_scaled, n_components, random_state=42, max_iter=200):
    gmm = GaussianMixture(
        n_components=n_components,
        covariance_type="full",
        random_state=random_state,
        max_iter=max_iter,
        n_init=10,
        reg_covar=1e-6,
    )
    gmm.fit(X_scaled)
    labels = gmm.predict(X_scaled)
    return gmm, labels

**A.2 RFM 特征工程（`core/dataset.py`）**

```python
def engineer_rfm_features(rfm_df, clip_bounds=None, fit_bounds=False):
    out = rfm_df.copy()
    bounds = dict(clip_bounds or {})
    for col in ("Frequency", "Monetary"):
        out[col] = np.log1p(out[col].astype(float).clip(lower=0))
    if fit_bounds:
        for col in ("Frequency", "Monetary"):
            bounds[col] = float(np.percentile(out[col], 99.0))
    for col in ("Frequency", "Monetary"):
        if bounds.get(col) is not None:
            out[col] = out[col].clip(upper=bounds[col])
    return out, bounds
```
